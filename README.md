# AgentBlocks

**Stand on the shoulders of giants. Build your own custom agents.**

AgentBlocks is an open research and building project for composable agents.

Instead of treating each agent as a closed black box, AgentBlocks breaks agents down into reusable building blocks, studies the best implementation of each block across the ecosystem, and turns those insights into a modular foundation for building custom agents.

The goal is not just to analyze existing agents. The goal is to make agent creation composable, understandable, and eventually accessible to everyone.

---

## Vision

We believe the future of agents is not one giant black-box system.

The future is composable.

Different agent systems are strong at different things:
- planning
- tool use
- execution loops
- context management
- memory
- interface design
- collaboration
- workflow automation
- safety and permissions
- multimodal interaction

AgentBlocks exists to identify the best building blocks across the ecosystem, understand why they work, and make them reusable for generating better custom agents for different people, teams, and companies.

---

## What AgentBlocks is

AgentBlocks is:

- An open research archive for agent systems
- A structured knowledge base for sources, notes, evidence, and conclusions
- A framework for deconstructing agents into reusable building blocks
- A foundation for composing custom agents from the best ideas in the ecosystem

---

## What AgentBlocks is not

AgentBlocks is **not**:

- A single all-in-one agent product
- Just a list of agent tools
- Just a benchmark leaderboard
- Just a personal notebook
- Just a content repository
- A static archive with no synthesis

This project is about turning fragmented agent innovation into reusable building blocks.

---

## Core idea

Most people evaluate agents as complete products.

We think that is only part of the picture.

A strong agent is usually made up of many different parts, and different systems are best at different parts. One agent may have the best planning loop. Another may have the best context handling. Another may have the best interaction design. Another may have the best execution environment.

So instead of asking:

> Which single agent is the best?

AgentBlocks asks:

> What are the core building blocks of an agent, and which system currently does each block best?

Once those building blocks are identified, studied, and abstracted, they can be recombined into custom agents designed for different goals and constraints.

---

## Core ideas

### 1. Agents should be deconstructed

Agents should not only be compared as whole products.  
They should also be broken down into their key building blocks.

### 2. Each building block can be studied independently

The best planning system may come from one agent.  
The best context management may come from another.  
The best interaction model may come from a third.

### 3. The best building blocks can be recombined

Once these blocks are understood, they can be reassembled into new agent systems that are better suited to specific users, teams, and workflows.

### 4. Agent creation should become accessible

In the long run, building an agent should feel less like reinventing a system from scratch and more like composing a system from the best available building blocks.

### 5. Research should lead to construction

This repository is not only for collecting information.  
It is meant to turn research into reusable foundations for building.

---

## Long-term goal

The long-term goal of AgentBlocks is to help create a world where anyone can build a custom agent by standing on the shoulders of the best existing systems.

That means:

- understanding agent systems deeply
- extracting reusable building blocks
- comparing implementations across the ecosystem
- preserving evidence and reasoning
- turning research into composable foundations
- making custom agent creation more open and more practical

Why this repository exists

The agent ecosystem is moving fast, but most of its knowledge is fragmented.

Valuable ideas are scattered across:

products
leaked code
open-source repos
demos
threads
blog posts
user observations
implementation details

Without a structured way to collect, compare, and abstract these ideas, most of that value stays trapped inside isolated systems.

AgentBlocks exists to turn scattered insights into reusable foundations.

Principles

This repository follows a few core principles:

Evidence over vague opinion

Important conclusions should be tied to sources, observations, or concrete implementation details whenever possible.

Structure over noise

The goal is not to collect everything. The goal is to collect and organize what matters.

Reusability over one-off insight

Research is most valuable when it can be reused for future analysis, writing, and building.

Composability over black boxes

We care less about worshipping a single agent and more about understanding how the best parts can be recombined.

Open foundation over closed myth

The future should not belong only to a few closed systems. It should become easier for more people to understand and build powerful agents.
